### 階段 2：學生管理（第 1-2 天）

#### 3.3 學生列表頁

`src/app/(admin)/admin/students/page.tsx`：
- 表格顯示所有學生（姓名、email、加入日期、最近登入、學習進度概覽）
- 「邀請新學生」按鈕（彈窗輸入 email + 姓名 → 寄邀請信）
- 每一行有「查看詳情」與「停用」按鈕
- 搜尋框可搜尋姓名或 email

#### 3.4 邀請學生流程

實作 Server Action `inviteStudent(email, name)`：
- 呼叫 Supabase Admin API 建立使用者
- 設定 role = 'student'
- 觸發發送邀請信
- 處理錯誤（email 已存在等）

#### 3.5 學生詳情頁

`src/app/(admin)/admin/students/[id]/page.tsx`：
- 顯示學生基本資料
- 學習進度統計（已學單字數、平均熟悉度、學習日曆）
- 最近 10 次測驗結果
- 「編輯資料」、「重設密碼」、「停用帳號」操作

---

### 階段 3：教材管理（第 2-3 天）

#### 3.6 單字管理頁

`src/app/(admin)/admin/vocabulary/page.tsx`：
- 表格欄位：假名、漢字、中譯、難度、標籤、建立日期、操作
- 搜尋框（搜假名或漢字或中譯）
- 篩選器：難度（N5-N1）、標籤
- 「新增單字」按鈕
- 每行有「編輯」、「刪除」按鈕（刪除有確認彈窗）
- 分頁（每頁 50 筆）

#### 3.7 單字新增/編輯表單

彈窗式表單，欄位：
- 假名（必填）
- 漢字（選填）
- 中譯（必填）
- 例句（日文）（選填）
- 例句（中譯）（選填）
- 音檔（選填，可上傳檔案到 Supabase Storage 或留空使用瀏覽器 TTS）
- 難度（下拉選單）
- 標籤（多選）

實作 Server Actions：`createVocab`、`updateVocab`、`deleteVocab`

#### 3.8 文法管理頁

`src/app/(admin)/admin/grammar/page.tsx`：
- 類似單字管理，但欄位為：句型、解說、例句、難度、標籤
- 例句採 `jsonb` 陣列格式，支援多筆例句
- 解說可使用 Markdown 編輯器（推薦 `@uiw/react-md-editor`）

#### 3.9 題目管理頁

`src/app/(admin)/admin/questions/page.tsx`：
- 表格欄位：題目、類型（選擇/填空/配對）、答案、難度、操作
- 篩選器：類型、難度、關聯單字/文法
- 新增題目時依題型動態切換表單：
  - **選擇題**：題目 + 4 個選項 + 正確答案
  - **填空題**：題目（用 ___ 表示空格） + 答案 + 寬鬆比對選項
  - **配對題**：左欄項目陣列 + 右欄項目陣列 + 正確配對關係
- 可選關聯到某個單字或文法

- 標籤分兩類：等級（N5/N4/N3/N2/N1）、主題（會話/商業/旅遊等）
- 標籤可設定顏色（用 colorPicker）
- CRUD 操作

### 階段 5：批次匯入（第 5 天）

#### 3.12 單字批次匯入

`src/app/(admin)/admin/import/page.tsx`：

**步驟 1：下載範本**
- 提供「下載 Excel 範本」按鈕
- ⚠️ **注意**：用戶會提供自己的 Excel 格式給您參考，請依照用戶的實際格式調整下方欄位定義
- 預設範本欄位：`假名 | 漢字 | 中譯 | 例句日文 | 例句中譯 | 難度 | 標籤（用逗號分隔）`

**步驟 2：上傳檔案**
- 接受 .xlsx、.csv 檔案
- 用 `xlsx` 套件解析

**步驟 3：預覽與校驗**
- 顯示解析後的表格
- 標示問題列（如假名/中譯空白、難度格式錯誤）
- 顯示「將匯入 N 筆，N 筆有誤」

**步驟 4：確認匯入**
- 點「確認匯入」執行批次寫入
- 顯示進度條
- 完成後顯示「成功匯入 N 筆，失敗 N 筆」

**步驟 5：失敗紀錄下載**
- 若有失敗，可下載失敗清單（含原因）

## 📋 F — Format（格式要求）

### UI 設計原則
- 採用 shadcn/ui 元件保持風格一致
- 表格用 `<Table>` 元件
- 表單用 `<Form>` + React Hook Form
- 彈窗用 `<Dialog>` 或 `<Sheet>`（手機友善）
- 通知用 `<Toast>` 顯示成功/失敗

### 互動格式
- 每完成一個頁面，請我打開實際測試後再進下一個
- 涉及 Server Action 時先說明該 action 在做什麼
- 出現複雜邏輯（如 RLS 權限）會解釋為何這樣寫

### 程式碼組織
- Server Actions 集中在 `src/app/actions/` 各自的子資料夾
- 共用 form schema 放 `src/lib/validations/`
- 大型表單元件獨立成檔

1. **用戶會提供 Excel 格式**：開始 3.12 前請主動詢問用戶提供格式，再依該格式調整匯入邏輯
