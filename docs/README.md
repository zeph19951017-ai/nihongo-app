## 🚀 使用流程

### Step 1：準備 Claude Code

1. 在電腦上安裝 Claude Code（若尚未安裝）
2. 建立空白資料夾，例如 `~/projects/nihongo-app`
3. 在該資料夾內開啟 Claude Code

### Step 2：開始 WP1

把 **兩份文件** 一起貼給 Claude Code：

```
1. 00_主控文件.md 全文
2. 01_WP1_環境建置.md 全文
```

開頭請告訴它：

> 我要開始 Phase 1 WP1，請依照附上的 Prompt 引導我完成。

### Step 3：完成 WP1 並驗收

依照 Prompt 中的「✅ T — Task 驗收清單」逐項打勾。
全部完成後保存 `WP1_完成報告.md`。

### Step 4：依序執行 WP2-WP6

每個 WP 都按照同樣流程：

```
1. 開啟新的 Claude Code 對話（避免 context 過長）
2. 貼上 00_主控文件.md
3. 貼上對應的 WP Prompt
4. 開頭說：「我已完成 WP[N-1]，現在開始 WP[N]」
5. 依照 Prompt 引導完成
6. 對照驗收清單打勾
```

## 💡 重要原則

### ✅ Do

- 每個 WP 完成才進下一個
- 每天 push 程式碼到 GitHub
- 遇到不確定的設計決策，回查主控文件
- 完成每個 WP 後實際測試所有驗收項目

### ❌ Don't

- 不要跳著做（例如先做測驗再做後台）
- 不要在一個 WP 沒完成時就進下一個
- 不要忽略驗收清單
- 不要修改主控文件中的「已決策項目」（除非有重大原因）

## 🆘 遇到問題怎麼辦

### 開發中卡關
- 回到對話請 AI 助手協助釐清
- 提供具體錯誤訊息與情境

### 想追加功能
- 不要立即在當前 WP 加入
- 記下來，等該 Phase 完成後一次規劃

### 想調整已決策項目
- 評估影響範圍（會不會影響資料庫 / 已寫的程式碼）
- 找 AI 助手討論再決定
