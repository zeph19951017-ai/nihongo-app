## 📌 C — Context（情境）

WP1-WP3 已完成，老師可建立教材並指派給學生。
本 WP 要建立**學生實際使用的學習介面**，是直接影響使用體驗的關鍵工作包。

關鍵需求：
- **沉浸式單字卡**：全螢幕、左右滑動切換、流暢動畫
- **間隔複習演算法**：用 SM-2 簡化版（會了 → 1天 → 3天 → 7天 → 14天）
- **手機優先**：學生主要在通勤時用手機學
- **行動裝置體驗極佳**：底部導覽列、手勢操作、流暢動畫

### 階段 2：沉浸式單字卡（第 2-3 天，本 WP 核心）

#### 4.3 安裝必要套件

```bash
npm install framer-motion @use-gesture/react
```

#### 4.4 單字卡列表入口

`src/app/(student)/study/vocabulary/page.tsx`：
- 顯示「今日要複習的單字（N 個）」
- 顯示老師指派的課程包列表（卡片式）
- 點擊任一課程包進入學習模式

#### 4.5 沉浸式學習頁

`src/app/(student)/study/vocabulary/[packId]/page.tsx`：

**設計規格**：
- **全螢幕模式**：隱藏導覽列、進入後狀態列也隱藏（用 viewport meta）
- **頂部進度**：「3 / 20」+ 細進度條
- **頂部右側**：關閉按鈕（離開學習）
- **中央卡片**：
  - 正面：大字顯示假名 + 漢字（漢字較小）
  - 卡片可上下翻面（點擊或上滑）
  - 背面：中譯 + 例句 + 例句中譯 + 發音按鈕
- **底部操作**：
  - 左：「不會」（紅色）
  - 中：「發音」（藍色按鈕，呼叫 Web Speech API）
  - 右：「會了」（綠色）
- **手勢**：
  - 左滑卡片：標記「不會」+ 滑動到下一張
  - 右滑卡片：標記「會了」+ 滑動到下一張
  - 點擊卡片：翻面
- **完成時**：顯示「今日學習完成！」+ 統計（會了 N、不會 N）+ 「再來一次」/「回首頁」按鈕

#### 4.6 卡片動畫實作要點

用 `framer-motion`：
- 卡片切換：水平滑動 + 淡入淡出（300ms）
- 翻面：3D rotate Y 軸（500ms）
- 滑動回饋：卡片跟隨手指移動，超過閾值（100px）才執行
- 滑動方向有顏色暗示：右滑顯示綠色光暈、左滑紅色

#### 4.7 發音功能

實作 `src/lib/utils/speech.ts`：
```typescript
export function speakJapanese(text: string) {
  const utterance = new SpeechSynthesisUtterance(text);
  utterance.lang = 'ja-JP';
  utterance.rate = 0.9;
  speechSynthesis.speak(utterance);
}
```

如果單字有 `audio_url` 則優先播放真人錄音，否則用 TTS。

### 階段 3：間隔複習演算法（第 3-4 天）

#### 4.8 實作 SM-2 簡化版

`src/lib/utils/spaced-repetition.ts`：

**演算法規則**：

| 當前 familiarity_level | 標記「會了」 → 下次複習 | 標記「不會」 → 下次複習 |
|----------------------|----------------------|----------------------|
| 0 (新單字)            | 1 天後                | 30 分鐘後             |
| 1                    | 3 天後                | 30 分鐘後             |
| 2                    | 7 天後                | 1 天後（降到 1）        |
| 3                    | 14 天後               | 3 天後（降到 2）        |
| 4                    | 30 天後               | 7 天後（降到 3）        |
| 5 (已掌握)            | 90 天後               | 14 天後（降到 3）       |

#### 4.9 整合演算法到單字卡

每次標記「會了/不會」時：
- 更新 `study_progress` 表
- 計算 `next_review_at`
- 更新 `familiarity_level` 與 `review_count`

#### 4.10 「今日待複習」邏輯

學生首頁與單字卡列表的「今日該複習」：
- 查詢 `study_progress` 表 `next_review_at <= now()` 的單字
- 加上「從未學過」的新單字（每天最多 10 個新單字）
- 排序：新單字優先 → familiarity 低的優先

## 📋 F — Format（格式要求）

### 動畫品質要求
- 60fps 流暢
- 用 `transform` 而非 `top/left`（避免 reflow）
- 用 `will-change` 提示瀏覽器（謹慎使用）
- 進場動畫 < 500ms，避免冗長

### 手機優先設計
- 觸控目標至少 44x44px
- 字級至少 16px（避免 iOS 自動放大）
- 重要按鈕放底部（拇指好按）
- 避免 hover-only 互動

### 程式碼組織
- 動畫元件獨立成檔
- 演算法邏輯放 `src/lib/utils/`
- 共用 hooks 放 `src/hooks/`
- 樣式優先用 Tailwind，複雜動畫才用 CSS module

## 🚧 注意事項

1. **iOS Safari 全螢幕**：iOS 不支援 Fullscreen API，要用 viewport meta + standalone 模式
2. **Web Speech API 相容性**：iOS Safari 需要使用者點擊後才能播放（瀏覽器限制）
3. **動畫效能**：低階手機可能掉幀，重要動畫加 `prefers-reduced-motion` 媒體查詢備案
4. **資料量**：學生若有上千單字，注意分批載入避免一次拉取太多
