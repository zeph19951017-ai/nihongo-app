請依照以下順序完成 9 個工作項目，**每完成一項先告訴我，等我確認再做下一項**：

### 1.1 確認 GitHub 帳號可用
- 請我提供 GitHub username
- 確認我是否已安裝 Git（指引指令：`git --version`）
- 若未安裝請給我 macOS / Windows 的下載連結

### 1.2 引導註冊 Vercel
- 請我前往 https://vercel.com/signup
- 用 GitHub 登入授權
- 確認儀表板可正常打開

### 1.3 引導註冊 Supabase
- 請我前往 https://supabase.com
- 用 GitHub 登入
- 引導我點選 "New project"
- 設定：
  - Project name: `nihongo-app`
  - Database password: 請我自訂並妥善保存
  - Region: 選擇 `Northeast Asia (Tokyo)`（離台灣最近）
- 等待建立完成（約 2 分鐘）

### 1.4 取得 Supabase 連線資訊
- 請我點選左側 Project Settings → API
- 取得 `Project URL` 與 `anon public key`
- 引導我把這兩個值記下來（後續會用）

### 1.5 初始化 Next.js 專案
請執行以下指令並逐一說明每個指令在做什麼：

```bash
# 在本地建立專案
npx create-next-app@latest nihongo-app

# 互動式選項建議：
# - TypeScript: Yes
# - ESLint: Yes
# - Tailwind CSS: Yes
# - src/ directory: Yes
# - App Router: Yes
# - Customize import alias: No (用預設 @/*)

cd nihongo-app
```

### 1.6 安裝核心套件

```bash
# Supabase 客戶端
npm install @supabase/supabase-js @supabase/ssr

# UI 元件庫
npx shadcn-ui@latest init
# 選擇 default style, slate base color, CSS variables: yes

# 常用工具
npm install lucide-react clsx tailwind-merge
```

### 1.7 設定環境變數
- 在專案根目錄建立 `.env.local`，內容：

```
NEXT_PUBLIC_SUPABASE_URL=（請填入 Supabase Project URL）
NEXT_PUBLIC_SUPABASE_ANON_KEY=（請填入 anon public key）
```

- 同時更新 `.gitignore` 確保 `.env.local` 不會被 commit

### 1.8 建立 Supabase 客戶端工具

在 `src/lib/supabase/` 建立兩個檔案：
- `client.ts`（瀏覽器端使用）
- `server.ts`（Server Component / Route Handler 使用）

請按 Supabase 官方 SSR 範例實作。

### 1.9 建立基本頁面架構並推上 GitHub + 部署 Vercel

修改 `src/app/page.tsx` 為簡單的歡迎頁：

```tsx
export default function Home() {
  return (
    <main className="flex min-h-screen items-center justify-center">
      <div className="text-center">
        <h1 className="text-4xl font-bold mb-4">日語教學 PWA</h1>
        <p className="text-gray-600">系統建置中...</p>
      </div>
    </main>
  );
}
```

接著：

```bash
# 初始化 Git
git init
git add .
git commit -m "Initial commit: Next.js + Supabase setup"

# 建立 GitHub repository
# 引導我用瀏覽器去 github.com/new 建立 repository，repo 名稱：nihongo-app

git push -u origin main

# 連到 Vercel 部署
# 引導我去 Vercel 點 "New Project"

```

完成後請我打開 Vercel 給的網址，確認看到「日語教學 PWA」歡迎頁。

### 1.10 建立資料夾結構

最後建立以下標準結構，並在每個資料夾放一個 `.gitkeep`：

```

├── app/
│   ├── (auth)/        # 登入註冊群組路由
│   ├── (admin)/       # 老師後台群組路由

│   └── api/           # API Routes
├── components/
│   ├── ui/            # shadcn/ui 元件
│   ├── admin/         # 老師後台元件
│   ├── student/       # 學生介面元件
│   └── shared/        # 共用元件
├── lib/
│   ├── supabase/      # Supabase 客戶端
│   ├── utils.ts       # 工具函數
│   └── types.ts       # TypeScript 型別定義
└── hooks/             # 自訂 React hooks
```

最後 commit + push，確認 Vercel 自動重新部署成功。

## 🚧 常見問題提前排雷

請主動提醒以下情境的處理方式：

1. **Node.js 版本問題**：Next.js 14 需要 Node 18+，先確認 `node --version`
2. **權限問題**：macOS/Linux 若 `npm install -g` 失敗，建議用 `npx` 或 `nvm`
3. **GitHub 推送失敗**：可能需要設定 SSH key 或 Personal Access Token
4. **Vercel 部署失敗**：通常是環境變數沒設定，引導我去 Vercel Dashboard 補上
5. **Supabase 連線失敗**：確認 URL 和 key 是否複製完整（沒有多空格）
