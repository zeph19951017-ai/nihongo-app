## 📌 C — Context（情境）

WP1-WP4 已完成，學生可學習單字與文法。
本 WP 要建立**測驗系統**，讓學生驗證學習成效，老師可看學生答題狀況。

關鍵需求：
- 三種題型：**選擇題、填空題、配對題**
- 三種挑題模式：**老師指派的課程包內隨機 / 學生自選範圍隨機 / 系統挑錯題優先**
- 填空題支援寬鬆比對（忽略全形/半形、忽略空格）
- 答題紀錄完整寫入 `quiz_results`，並產生 `session_id` 串連同一次測驗
- 老師端可看每位學生的測驗統計

## ⚙️ A — Action（行動）

### 階段 1：測驗起始流程（第 1 天）

#### 5.1 測驗入口頁

`src/app/(student)/quiz/page.tsx`：
- 顯示三個大卡片，對應三種模式：
  1. **🎯 課程包測驗**：選擇老師指派的某個課程包
  2. **📚 範圍測驗**：選擇難度（N5-N1）+ 標籤組合
  3. **🔥 錯題重練**：自動挑出最近答錯過的題目
- 卡片下方顯示該模式可用的題目數

#### 5.2 測驗設定頁

`src/app/(student)/quiz/start/page.tsx`：
- 依模式顯示不同設定：
  - **課程包**：選擇課程包（下拉選單）
  - **範圍**：選擇難度（多選）+ 標籤（多選）
  - **錯題**：自動帶入錯題池
- 共同設定：
  - 題數（10 / 20 / 30 / 50）
  - 題型偏好（全部 / 只選擇題 / 只填空 / 只配對）
- 「開始測驗」按鈕

#### 5.3 挑題邏輯實作

`src/lib/quiz/picker.ts`：

```typescript
async function pickQuestions({
  mode: 'pack' | 'range' | 'mistakes',
  packId?: string,
  difficulties?: string[],
  tagIds?: string[],
  count: number,
  questionTypes?: string[]
}): Promise<Question[]>
```

**邏輯**：
- `pack` 模式：從 `lesson_packs.question_ids` 隨機抽
- `range` 模式：依條件查 `questions` 表
- `mistakes` 模式：從 `quiz_results` 找出 `is_correct = false` 的最近 100 題，隨機抽
- 確保抽出的題目不重複
- 若可用題目少於要求數，回傳實際數量並提示

### 階段 2：作答介面（第 1-2 天）

#### 5.4 測驗主畫面

`src/app/(student)/quiz/[sessionId]/page.tsx`：

**全螢幕設計（類似單字卡）**：
- 頂部：題目編號（5/20）+ 進度條 + 計時器（選做）+ 退出按鈕
- 中央：題目卡片（依題型動態渲染）
- 底部：「下一題」按鈕（答完才能按）

**狀態管理**：
- 用 `useReducer` 管理測驗狀態（避免太多 state）
- 狀態包含：當前題號、所有題目、使用者答案陣列、開始時間
- 進入測驗時建立 `session_id`（uuid）

#### 5.5 選擇題元件

`src/components/student/quiz/MultipleChoiceQuestion.tsx`：
- 顯示題目
- 4 個選項，垂直排列（手機友善）
- 點選項後立即顯示對錯：
  - 答對：選項變綠 + 「答對了！✓」
  - 答錯：所選變紅 + 正確答案變綠 + 「正確答案是 X」
- 顯示「下一題」按鈕

#### 5.6 填空題元件

`src/components/student/quiz/FillBlankQuestion.tsx`：
- 題目中的 `___` 換成 input 欄位
- 支援多個空格
- 學生輸入後按「送出」
- 寬鬆比對邏輯：
  - 忽略前後空格（trim）
  - 忽略全形/半形差異（用 `normalize`）
  - 忽略大小寫
- 答對顯示綠色，答錯顯示正確答案（紅色）

實作 `src/lib/quiz/answer-checker.ts`：
```typescript
export function checkAnswer(
  userAnswer: string,
  correctAnswer: string,
  options?: { caseSensitive?: boolean; strictWidth?: boolean }
): boolean
```

#### 5.7 配對題元件

`src/components/student/quiz/MatchingQuestion.tsx`：
- 左右兩欄，每欄 4-6 個項目
- 學生點擊配對：
  - 點左側某項 → 變黃色（已選中）
  - 再點右側某項 → 連起來，畫一條線
  - 或拖拉連線（推薦 `react-archer` 套件）
- 全部配對完才能送出
- 送出後顯示對錯：對的綠線、錯的紅線 + 正確配對虛線

- 離線時暫存到 IndexedDB

## 📋 F — Format（格式要求）

### 互動格式
- 每完成一個題型，請我實際做一次測驗測試
- 涉及挑題演算法時主動說明邏輯
- 寬鬆比對的 case 主動列舉測試（例如「ABC」「ＡＢＣ」「abc」應視為相同）

### 程式碼品質
- 三種題型用相同介面（type Question），方便擴充
- 判分邏輯獨立成 pure function 易測試
- 使用 React Hook Form 管理表單狀態

### UX 注意
- 答題後不允許修改（避免作弊）
- 退出測驗有確認彈窗（避免誤觸）
- 計時器選做（可能造成壓力，預設關閉）

## 🚧 注意事項

1. **配對題的線條繪製**：在響應式環境下重新計算位置較複雜，可先用點選版本（點左+點右配對），未來再加拖拉
2. **題目重複問題**：同一次測驗確保不重複，跨次測驗允許重複
3. **錯題池維護**：「最近答錯」定義為過去 30 天答錯，避免錯題池無限累積
4. **離線答題暫存**：本 WP 先實作介面，IndexedDB 暫存留到 WP6 完整實作
