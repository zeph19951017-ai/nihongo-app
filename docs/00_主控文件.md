## 🛠️ 二、技術架構

### 技術選型

| 層 | 技術 | 為何選此 |
|----|------|---------|
| **前端框架** | Next.js 14+ (App Router) | 業界主流、SSR/SEO 好、Vercel 一鍵部署 |
| **UI 框架** | TailwindCSS + shadcn/ui | 開發快、響應式、AI 維護容易 |
| **後端服務** | Supabase | 帳號/資料庫/儲存/即時同步一站式、免費方案足夠 |
| **資料庫** | PostgreSQL（Supabase 內建） | 支援複雜關聯、RLS 權限規則 |
| **身分驗證** | Supabase Auth | 內建 email 邀請信流程 |
| **部署** | Vercel | 免費、自動部署、CDN 加速 |
| **音檔來源** | 瀏覽器 Web Speech API + 老師上傳 | 免費、不依賴外部 API |
| **PWA** | next-pwa 套件 | Service Worker / Cache / IndexedDB |
| **離線資料庫** | IndexedDB（透過 Dexie.js 套件） | 離線暫存教材、答題紀錄 |

### 架構流程

```
[老師 / 學生]
    ↓ 透過瀏覽器或 PWA 圖示
[Next.js 前端 (部署於 Vercel)]
    ↓ API 呼叫
[Supabase 後端服務]
    ├── PostgreSQL 資料庫
    ├── Auth 身分驗證
    └── Storage 檔案儲存（音檔、圖片）
```

## 🗃️ 五、資料庫 Schema（8 張資料表）

### 5.1 `users`（使用者表）
```sql
- id (uuid, primary key)
- email (text, unique, not null)
- name (text, not null)
- role (text, 'teacher' | 'student')
- created_at (timestamp)
- updated_at (timestamp)
```

### 5.2 `vocabulary`（單字表）
```sql
- id (uuid, primary key)
- kana (text, not null)            -- 假名
- kanji (text, nullable)            -- 漢字
- meaning (text, not null)          -- 中譯
- example_jp (text, nullable)       -- 例句（日文）
- example_zh (text, nullable)       -- 例句（中譯）
- audio_url (text, nullable)        -- 音檔網址
- difficulty (text, 'N5'|'N4'|'N3'|'N2'|'N1')
- created_by (uuid, references users.id)
- created_at (timestamp)
```

### 5.3 `grammar`（文法句型表）
```sql
- id (uuid, primary key)
- pattern (text, not null)          -- 句型
- explanation (text, not null)      -- 解說
- examples (jsonb)                  -- 例句陣列
- difficulty (text)
- created_by (uuid, references users.id)
- created_at (timestamp)
```

### 5.4 `questions`（題庫表）
```sql
- id (uuid, primary key)
- type (text, 'multiple_choice'|'fill_blank'|'matching')
- question (text, not null)
- options (jsonb, nullable)         -- 選擇題選項
- correct_answer (text, not null)
- related_vocab_id (uuid, nullable, references vocabulary.id)
- related_grammar_id (uuid, nullable, references grammar.id)
- difficulty (text)
- created_by (uuid, references users.id)
- created_at (timestamp)
```

### 5.5 `lesson_packs`（課程包表）
```sql
- id (uuid, primary key)
- name (text, not null)
- description (text)
- vocab_ids (uuid[])                -- 包含的單字 IDs
- grammar_ids (uuid[])              -- 包含的文法 IDs
- question_ids (uuid[])             -- 包含的題目 IDs
- created_by (uuid, references users.id)
- created_at (timestamp)
```

### 5.6 `study_progress`（學習進度表）
```sql
- id (uuid, primary key)
- user_id (uuid, references users.id)
- vocab_id (uuid, references vocabulary.id)
- familiarity_level (int, 0-5)      -- 熟悉度
- next_review_at (timestamp)        -- 下次複習日
- review_count (int)                -- 複習次數
- updated_at (timestamp)
```

### 5.7 `quiz_results`（答題紀錄表）
```sql
- id (uuid, primary key)
- user_id (uuid, references users.id)
- question_id (uuid, references questions.id)
- is_correct (boolean)
- user_answer (text)
- answered_at (timestamp)
- session_id (uuid)                 -- 同一次測驗的 ID
```

### 5.8 `tags`（標籤表）
```sql
- id (uuid, primary key)
- name (text, not null)
- type (text, 'level'|'topic')      -- N5/會話/商業 等
- color (text, nullable)            -- 顯示顏色
- created_at (timestamp)
```

### 關聯標籤表（多對多）

```sql
-- vocabulary_tags
- vocab_id (uuid)
- tag_id (uuid)
PRIMARY KEY (vocab_id, tag_id)

-- grammar_tags
- grammar_id (uuid)
- tag_id (uuid)
PRIMARY KEY (grammar_id, tag_id)
```

## 🔐 六、權限規則（RLS Row Level Security）

### 老師（role='teacher'）
- ✅ 可讀寫所有 `users`、`vocabulary`、`grammar`、`questions`、`lesson_packs`、`tags`
- ✅ 可讀所有 `study_progress`、`quiz_results`（看學生進度用）
- ❌ 不可寫他人的 `study_progress`、`quiz_results`

### 學生（role='student'）
- ✅ 可讀（不可寫）所有 `vocabulary`、`grammar`、`questions`、`lesson_packs`、`tags`
- ✅ 可讀寫自己的 `study_progress`、`quiz_results`
- ❌ 不可讀其他學生的進度
- ❌ 不可寫教材資料

## 🎨 七、UI 設計原則

- **行動優先**：先確保手機體驗佳，再適配平板/桌面
- **沉浸式單字卡**：全螢幕 + 左右滑動 + 流暢動畫
- **一致風格**：統一使用 shadcn/ui 元件，配色簡約
- **無障礙**：字級可調、對比度足夠、支援鍵盤導航
- **回饋明確**：每個操作都有即時視覺回饋（loading / success / error）

## 🚀 九、執行流程

### 您的工作流程（依序執行）

1. **準備環境**：開啟 Claude Code，建立空白專案資料夾
2. **依序執行 6 個 WP**：
   - 把 `主控文件.md` + `WP1 Prompt` 一起貼給 Claude Code
   - 完成 WP1 後驗收（依 SMART 目標檢查）
   - 把 `主控文件.md` + `WP2 Prompt` 貼給 Claude Code 接著做
   - 以此類推，直到 WP6 完成
3. **上線後試用**：邀請 1-2 位學生試用 1 週，收集回饋
4. **進入 Phase 2**：根據回饋微調，並開啟 Phase 2 工作包

### 重要原則

- ⚠️ **每個 WP 完成才進下一個**，不要跳著做
- ⚠️ **每個 WP 完成後立即驗收**，問題早發現早解決
- ⚠️ **每天 push 到 GitHub**，確保有版本紀錄
- ⚠️ **遇到不確定的設計決策**，回來查主控文件，避免歪掉
