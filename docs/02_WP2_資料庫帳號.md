## ⚙️ A — Action（行動）

### 階段 1：資料庫建置（第 1-2 天）

#### 2.1 建立 SQL migration 檔案

在專案根目錄建立 `supabase/migrations/` 資料夾，按執行順序建立 SQL 檔：

**01_create_users_table.sql**
```sql
-- 使用者表，與 Supabase auth.users 連動
CREATE TABLE public.users (
  id uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email text UNIQUE NOT NULL,
  name text NOT NULL,
  role text NOT NULL CHECK (role IN ('teacher', 'student')) DEFAULT 'student',
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

-- 自動更新 updated_at
CREATE TRIGGER update_users_updated_at BEFORE UPDATE ON public.users
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

依序建立其他 7 張表的 SQL（vocabulary / grammar / questions / lesson_packs / study_progress / quiz_results / tags）以及兩張關聯表（vocabulary_tags / grammar_tags）。

每個 SQL 檔案請：
- 加註中文註解說明欄位用途
- 加上必要的索引（INDEX）提升查詢效能
- 加上必要的 CHECK 約束確保資料品質

#### 2.2 建立 RLS 權限規則

建立 `supabase/migrations/10_rls_policies.sql`：

針對每張表寫 RLS policy，依照主控文件第六節的權限規則：
- 老師（role='teacher'）的權限
- 學生（role='student'）的權限

請特別注意：
- 教材表（vocabulary、grammar、questions、lesson_packs、tags）：老師可讀寫，學生只可讀
- 進度表（study_progress、quiz_results）：學生只能讀寫自己的，老師可讀全部
- users 表：使用者可讀寫自己的資料，老師可讀全部

#### 2.3 在 Supabase Dashboard 執行 SQL

引導我在 Supabase Dashboard → SQL Editor 依序執行所有 SQL，並驗證：
- 8 張表 + 2 張關聯表都建立成功
- 在 Table Editor 可看到正確欄位
- 在 Authentication → Policies 可看到 RLS 規則生效

#### 2.4 產出 TypeScript 型別定義

執行 Supabase CLI 自動產出型別：

```bash
npx supabase gen types typescript --project-id [project-ref] > src/lib/types/database.ts
```

確認型別檔正確匯出，並在 `src/lib/supabase/client.ts` 套用型別。

### 階段 2：身分驗證頁面（第 3 天）

#### 2.5 開發登入頁

建立 `src/app/(auth)/login/page.tsx`：
- email + 密碼輸入欄位
- 表單驗證（用 React Hook Form + Zod）
- 「忘記密碼？」連結
- 登入成功後依角色跳轉：teacher → `/admin`、student → `/dashboard`
- 登入失敗顯示友善錯誤訊息（中文）

#### 2.6 開發學生註冊頁（從邀請連結進入）

建立 `src/app/(auth)/setup-password/page.tsx`：

- 密碼強度要求：至少 8 字元、含英文+數字
- 設定成功後自動登入並跳轉 `/dashboard`

#### 2.7 開發忘記密碼頁

建立 `src/app/(auth)/forgot-password/page.tsx`：
- 輸入 email
- 觸發 Supabase 發送密碼重設信
- 顯示「重設密碼信已發送，請查收」

建立 `src/app/(auth)/reset-password/page.tsx`：
- 輸入新密碼 + 確認
- 重設成功後跳轉登入頁

#### 2.8 開發登出功能

在 `src/components/shared/` 建立 `LogoutButton.tsx`：
- 點擊呼叫 `supabase.auth.signOut()`
- 跳轉回 `/login`

### 階段 3：路由保護（第 4 天）

#### 2.9 建立 middleware.ts

在專案根目錄建立 `middleware.ts`：
- 檢查 user session
- 未登入訪問 `/admin/*` 或 `/dashboard/*` → 跳轉 `/login`
- 已登入訪問 `/login` → 跳轉對應角色首頁
- 學生訪問 `/admin/*` → 跳轉 `/dashboard`（403 邏輯）

#### 2.10 建立 useUser hook

在 `src/hooks/useUser.ts`：
- 封裝取得當前使用者資訊
- 包含 role 判斷
- 提供 `isTeacher` / `isStudent` 便利屬性

#### 2.11 建立第一個老師帳號

引導我在 Supabase Dashboard：
1. Authentication → Users → Add user → 建立我的帳號
2. SQL Editor 執行：
```sql
UPDATE public.users
SET role = 'teacher', name = '[您的名字]'
WHERE email = '[您的 email]';
```

確認登入後可進入 `/admin`。

## 📋 F — Format（格式要求）

### 互動格式
- SQL 執行前請先說明該 SQL 在做什麼
- 每階段結束時做一次小總結 + 等我確認
- 若涉及安全性（RLS、密碼），多解釋為什麼這樣設計

### 程式碼格式
- 所有 SQL 加中文註解
- 表單元件使用 shadcn/ui 的 Form / Input / Button
- 錯誤訊息用中文，但 console.log 用英文方便除錯
- 表單驗證 schema 集中放在 `src/lib/validations/`

### 安全注意事項
- 永不把 service_role key 放到前端
- 所有寫入操作走 Server Action 或 Route Handler
- 密碼欄位 type="password" + autocomplete 屬性
- 防止 SQL injection（用 Supabase SDK 而非拼接 SQL）

## 🚧 常見問題排雷

1. **RLS 太嚴格導致連自己都讀不到**：通常是 `auth.uid()` 用法錯誤
2. **邀請信沒收到**：檢查垃圾信箱、確認 Supabase Auth 的 SMTP 設定
3. **middleware 無限重導**：通常是 cookie 處理錯誤
4. **TypeScript 型別不對**：忘記重新產出 database.ts
